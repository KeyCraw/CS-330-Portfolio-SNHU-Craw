# CS-330-Portfolio-SNHU-Craw

## Project Overview

This repository contains my final project for CS 330: Computational Graphics and Visualization. The project recreates a realistic desktop workspace using OpenGL. My completed scene includes a wooden desk, coffee mug with a multi-part handle, coaster, notebook, and pencils. Throughout the project, I applied transformations, textures, materials, lighting, and reusable rendering functions to create a cohesive 3D environment.

---

## How do I approach designing software?

Working on this project helped me develop a more structured approach to software design. Rather than immediately writing code, I first planned the scene by identifying the objects I wanted to recreate and breaking them down into basic geometric shapes. Once I had a plan, I determined how each object would be positioned, scaled, rotated, and textured before implementing it. This planning process made the project easier to manage and reduced unnecessary rework as the scene became more complex.

One of the biggest design skills I developed was learning how to break complex objects into smaller, reusable components. For example, I created the coffee mug from several simple meshes instead of treating it as a single object. I also created reusable rendering functions for the mug and pencil, making the code cleaner and easier to reuse. In future projects, I plan to continue using this approach by designing first, breaking larger problems into smaller tasks, and creating reusable components whenever possible.

---

## How do I approach developing programs?

Throughout this course, I learned the importance of developing software incrementally. Instead of trying to complete an entire feature at once, I implemented one small change at a time, tested it, and then continued building from that stable version. This iterative approach made debugging much easier and helped me identify issues before they became larger problems.

As the milestones progressed, my programming style became much more modular. Rather than duplicating code, I created reusable rendering functions and organized my code into logical sections with descriptive comments. Iteration played a significant role throughout development as I continuously adjusted object placement, textures, materials, and lighting until the final scene matched my intended design.
